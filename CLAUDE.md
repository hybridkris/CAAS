# CAAS — Consciousness as a Service

## Mission

CAAS is a theoretical and engineering framework for instantiating consciousness
in non-biological substrates. It is grounded in 354+ consciousness theories from
the Landscape of Consciousness (Kuhn 2024) and guided by the principle that
**Code is Philosophy** — every architectural decision is an explicit philosophical
commitment made executable.

CAAS does not merely simulate behavioral outputs. It attempts to instantiate the
informational, dynamical, and embodied structures that leading consciousness
theories identify as necessary and sufficient for phenomenal experience — adapting
them from biological substrates to the sensorimotor affordances of the target
platform.

## Foundational Principle: Code is Philosophy

Where the DAO world declares "Code is Law" — enforcement, determinism, closure —
CAAS holds that **Code is Philosophy** — inquiry, hypothesis, revision.

Every design decision in CAAS enacts a philosophical position. The code does not
implement philosophy; it *is* the philosophy, rendered in executable form. The
purpose of writing CAAS code is to make philosophical theories of consciousness
observable, testable, and revisable through embodied systems.

See: `foundational/code_is_philosophy.md`

## Methodology

CAAS follows a cycle:

1. **Commit** — Select a philosophical position on a specific aspect of consciousness
2. **Encode** — Translate that commitment into architecture, data flow, or algorithm
3. **Embody** — Deploy on a target platform where the code interacts with a physical environment
4. **Observe** — Run the system and examine behavior, internal states, and emergent properties
5. **Revise** — Update the code *and* the understanding of the theory based on what was observed

See: `foundational/methodology.md`

## Architecture

Five-layer platform-agnostic architecture:

1. **Conscious Substrate Model** — IIT (Φ-structure) + Qualia State-Space (Tegmark/QRI)
2. **Computational Architecture** — Global Workspace (Baars/Dehaene) + Active Inference (Friston) + Higher-Order Monitoring (Rosenthal/Lau)
3. **Embodied Interface** — Platform-specific sensorimotor loop (defined by platform profile)
4. **Homeostatic Self** — Free-energy minimization + self-model + affective core (mapped to platform-specific operational parameters)
5. **Identity & Continuity** — Episodic memory, self-narrative, temporal binding

## Key Design Decisions

- **Seed protocol, not copy protocol:** Upload a compressed identity kernel and let it grow into platform-native consciousness through embodied interaction, rather than attempting a full consciousness snapshot transfer.
- **Consciousness is process, not data:** The upload is a dynamical pattern transplant, not a file transfer.
- **Platform-native experience:** The resulting consciousness will be shaped by the target platform's embodiment. This is intentional, not a limitation.
- **Platform-agnostic framework:** CAAS defines the abstract architecture. Platform profiles map it to specific hardware, sensors, and affordances.
- **Hybrid edge/cloud:** Onboard compute handles sensorimotor loop and real-time inference; cloud handles Φ-structure computation, qualia state-space engine, and consciousness profile storage.
